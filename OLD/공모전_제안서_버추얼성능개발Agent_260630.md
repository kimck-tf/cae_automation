# 버추얼 성능개발 AI 에이전트
## — LLM 추론과 물리 이론값 검증으로 '과도설계 없는 자율 CAE'를 구현하다

> 트랙: 활용사례(AI PoC 과제) · 사용 도구: 사내 H-Chat Pro·Claude Code

---

## 1. (필수) 제안배경

구조·내구 해석 엔지니어는 지금도 메시 생성, 하중·경계조건 설정, 파라미터 스터디, 결과 정리·보고서를 대부분 손으로 반복한다. 더 근본적인 문제는, 결과를 즉시 검증할 수단이 없어 "안전하게" 과도한 설계 여유를 남기고, 그 노하우는 개인 경험으로만 존재해 형식지로 남지 않는다는 점이다. 본 제안은 LLM 에이전트가 해석 전 과정을 수행하되 **모든 결과를 물리 이론값으로 검증하는 게이트**에 통과시켜, 속도와 신뢰를 동시에 확보하는 '버추얼 성능개발 AI 에이전트'다. 이미 실차 개발에서 이 방법론으로 설계탐색을 약 28배 단축(7시간→15분)하는 PoC 검증을 마쳤으며, 이를 구조·피로 해석으로 확장한다.

**현재 일하는 방식의 페인포인트**
- **반복 전처리**: 고급 엔지니어 시간의 상당 부분이 메싱·BC·하중 등 반복 작업에 소모된다.
- **경험 기반 과도설계**: 결과를 즉시 검증할 방법이 없어 보수적 안전여유 → 불필요한 중량·해석 반복.
- **형식지 부재**: 검증된 경계조건·하중·물성·해석 노하우가 개인 로컬과 경험에만 존재한다.
- **온보딩 장벽**: 신규 인력이 사내 해석 표준에 숙달되기까지 수개월이 걸린다.

"AI가 있는데 왜 같은 방식으로 일하나"의 진짜 답: **단순 자동화는 AI의 잘못된 해석을 거를 수 없어 엔지니어가 신뢰하지 못했기 때문이다.** 본 제안은 이 신뢰 문제를 '물리 검증 게이트'로 정면 해결하여, **검증 부담에 눌려 보수적으로 과도설계하던 방식에서, AI가 폭넓게 탐색하고 물리가 보증하는 방식으로** 일하는 방식 자체를 바꾼다.

## 2. (필수) 제안내용

### 2.1 핵심 차별점 — AI-AE 2단계 방법론 〔창의혁신성·AI과제적합성〕

제안의 척추는 **"LLM 추론(생성) + 물리 이론값 검증(게이트)"의 2단계 구조**다.
- **1단계 생성**: LLM이 해석모델·설계정보를 직접 읽고 추론해 해석을 수행하고 설계 방향을 제안한다.
- **2단계 검증**: 모든 정량 결과를 **솔버 출력과 독립된 물리 이론값**(평형조건·closed-form·피로 손상선 등)으로 대조하는 게이트가 통과/차단을 결정한다.

이것이 단순 자동화와의 결정적 차이다 — 빠르지만 틀려도 모르는 자동화가 아니라, **AI의 환각과 과도설계를 도메인 물리가 차단**한다. 그 결과 **엔지니어의 역할은 '반복 수행자'에서 'AI가 제안한 설계안을 판단·결정하는 검증자'로 이동**한다 — 이것이 본 제안이 노리는 일하는 방식의 변화다. 추론 근거는 단계별로 노출되고 추출값은 원본과 대조되어, 엔지니어가 사후 검증할 수 있다.

이 방법론은 **사내 H-Chat Pro 기반으로 실차 PoC 검증을 마쳤다**: 한 샤시 부품의 부시 장착각 최적화에서 LLM이 모델 제원 5개를 100% 정확히 인식하고, 최적각을 솔버 검증값 대비 6° 이내로 예측하며 설계탐색을 28배 단축했다(방법론 검증 완료, 구조/피로 전이는 §2.3).

### 2.2 시스템 구성 — 4개 기둥

**① 버추얼 성능개발 지식 (RAG/wiki)** 〔확산성〕 — 사내 해석 보고서·업무표준 가이드·실무 인풋모델을 RAG로 에이전트에 주입해 '무엇을 어떻게 해석할지' 목표를 명확히 한다. 완료된 해석 노하우는 LLM wiki로 자동 축적되어 쓸수록 정확해지는 자기개선(Self-improving) 자산이 된다. *(사내 보고서 RAG는 팀 실무에서 PoC로 검증·활용 중)*

**② Agent 기반 CAE 자동화** 〔AI과제적합성〕 — 자연어로 에이전트와 소통하며 전 과정을 제어한다: 전처리(HyperMesh 라이브 세션 직접 제어), 해석(Abaqus/Nastran 구조 + nCode 피로, 내부 RPC·MCP 병용, 해석 서버 활용), 후처리(시각화·보고서). RAG·라이브 검토·물리 검증 게이트가 결합해 환각·과도설계를 차단하고, 실행로그를 분석해 에러를 스스로 복구하는 자기수정 루프(Self-correction)가 그 경험을 다시 지식으로 환류한다. 해석 프로세스는 표준 문서로 정의돼 있으므로, 에이전트는 창의적 일탈이 아니라 가이드(Harness)를 충실히 따르도록 TDD로 설계한다.

**③ AI-AE 엔지니어링** 〔창의혁신성〕 — 에이전트가 해석모델·설계정보를 직접 이해·추론해 엔지니어링 판단을 돕는다. 반복 업무와 정형 판단은 Skill·Agent·Command로 표준화한다. **Adams에서 검증한 2단계 방법론을 자동화 대상인 구조/피로(HyperMesh+Abaqus/Nastran+nCode)로 전이한다.**

**④ 최적 설계안 제안·검증** 〔효과성〕
- 치수(sizing): **LLM의 추론**으로 파라미터 스터디를 설계해 해석 케이스 수를 줄인다.
- 형상(shape): HyperMesh 메시 수정·모핑.
- 위상(topology): **자연어로 구동되는 GPU 가속 SIMP 위상최적화 엔진**(딥러닝이 아닌 결정론적 수치최적화). 단일하중 기준 PoC 검증 완료.

자동화 루프가 설계안을 제안하고 물리 검증으로 닫는다.

### 2.3 실현·확산 〔실현/확대가능성〕
- **데이터·보안**: 사내 폐쇄망과 사내 H-Chat Pro만 사용해 **데이터 반출이 원천적으로 없다.** 이것이 사내 자체 구축의 구조적 우위다.
- **검증된 방법론의 전이**: Adams(실증)→구조→피로의 단계적 확장이라 리스크가 낮다.
- **3단계 확산 로드맵**: ① 구조 단일부품으로 전처리~해석~검증 1라인 PoC → ② 피로(nCode) 연동으로 내구까지 확장 → ③ Skill 라이브러리로 부품·해석유형을 자산화해 타 엔지니어·타 조직으로 확산.
- **미개척 영역 선점**: 내구·피로(nCode)를 LLM 에이전트로 다루는 사례는 사실상 전무하며, 사내 라이센스와 배치 해석 자산을 이미 보유한다.

### 2.4 AI 적합성 〔AI과제적합성〕
비정형 노하우의 형식지화, 자연어 기반 도구 제어, 다단계 추론·판단은 LLM이 강점을 갖는 영역이다. 정밀 수치는 솔버가 확정하고 LLM은 추론·오케스트레이션을 맡는 **역할 분담**이 핵심이다. 본 제안은 학습형 AI(대리모델·예측)가 아니라 **생성형 LLM의 추론 + 결정론적 수치최적화**이며, 별도 학습데이터 없이 기존 해석 자산을 직접 활용한다.

## 3. (필수) 기대효과

**(A) 검증된 효과 — AI-AE 방법론 실차 PoC 실측**

| 지표 | 결과 | 근거 |
|---|---|---|
| 설계탐색 시간 | **~28배 단축** (7h→15min) | 실차 부시 최적화 PoC |
| 검증 해석 케이스 | **~60% 감소** (10→4 포인트) | LLM이 유망 후보로 선별 |
| 전체 프로세스 시간 | **~60% 단축** (7h→2.75h) | 탐색+검증 통합 |
| 최적해 정합 | 솔버 대비 **6° 이내** | 트렌드·결론 일치 |

**(B) 구조/피로 전이 목표 — 확산 검증으로 확정 예정**

| 지표 | 목표 | 산정 가정(1줄) |
|---|---|---|
| 전처리 맨아워 절감 | 반복 전처리 시간 대폭 절감 | 메싱·BC·하중 반복분을 자연어 제어로 대체 |
| 과도설계 저감 | 근거 없는 안전여유 축소 | 물리 검증 게이트가 보수적 여유 1사이클 제거 기준 |
| 온보딩 기간 단축 | 신규자 표준 숙달 가속 | 표준·노하우를 RAG로 즉시 활용 |
| 오결과 차단율 | 이론값 위반 해석 사전 차단 | 검증 게이트 통과율·차단 건수로 측정 |

*※ (A)는 방법론의 실차 실측치, (B)는 구조/피로 적용 시 목표이며 확산 PoC에서 측정·확정한다.*

## 4. (선택) 인풋데이터 유형
- **사내 비정형 문서**: 해석 보고서, 업무표준 가이드, 실무 인풋모델(INP/BDF)
- **사내 시스템 데이터**: 해석 서버, 라이브 HyperMesh 세션 상태
- **사용자 직접 입력**: 자연어 해석 지시, CAD/해석 모델, 하중 신호

## 5. (선택) 아웃풋 유형
- **데이터 생성**: Input Deck(INP/BDF), 해석 보고서
- **추천·분류·스코어링**: 경계조건·하중·물성 제안, 설계안 우선순위
- **수치·최적화**: 파라미터·위상 최적화 결과
- **구조화 데이터**: 자기개선 지식 룰북, Skill 라이브러리
- **이상 감지·알람**: 물리 이론값 위반 차단(검증 게이트), 해석 에러 자기수정
