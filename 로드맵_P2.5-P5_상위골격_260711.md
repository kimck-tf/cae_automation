# cae-copilot 상위 로드맵 골격 — P2.5 → P3 · P4 · P-GUI → P5

> **지위:** SDD §13(페이즈 정의)의 **실행 관점 상세화 골격**이다. 페이즈 정의의 SSOT는 SDD이며 충돌 시 SDD가 우선한다. 본 문서는 각 페이즈의 입구·출구·덩어리·리스크를 **골격 수준**으로 고정해 페이즈 간 경계 침범을 막는 용도다 — 태스크 단위 상세는 §13 분할 원칙대로 **각 페이즈 직전에 별도 구현계획으로 작성**한다(P0~P2 선례). 골격의 제안 사항은 상세계획 수립 시 확정하고, 확정분 중 설계 변경은 SDD 개정노트로 환류한다.
>
> 작성: 2026-07-11 (Fable 상위 계획 세션). 전제 상태: P0·P1 완료(249 tests), P2 계획 확정(리뷰 F1~F8 반영판 `fdb1fd3`), P2 실행 대기.

## 1. 전체 의존 그래프

```
                 ┌─→ P2.5 실기 게이트 ──→ P3 사내 이관·실측 ──┐
P2 (Mock E2E =   │                                          ├─→ P5 성숙
    dev 프로파일) ├─→ P4 트랙② (dev 구간 — 실검증은 P3 환경 편승) ┤     (개방은 데이터 조건부)
                 └─→ P-GUI (병행) ──────────────────────────┘
                                    (P5 전제 = P3 실측 + P-GUI 완료)
```

- **직렬 축(주경로):** P2 → P2.5 → P3. 실기(HM)와 사내 환경이 각각의 게이트다.
- **병렬 축:** P4·P-GUI는 P2 완료 후 어느 시점이든 착수 가능(SDD §13). 단 P4의 실검증(nCode·실솔버)과 P-GUI의 운영 배선(H-Chat)은 P3 환경에 편승한다.
- **부유 인간 액션(페이즈 밖):** ① MANUAL_P1 A·B·E·F — 지금~P2.5 전 아무 때나(P2 Task 10 SubagentStop 실측과 같은 실기 세션으로 묶기 권장) ② MANUAL_P2 A~D — P2 중.

3. 구현·하드닝 반영 후 `CHECKSUMS.sha256` 재생성(이후 이것이 동결 기준), `.gitattributes -text` 유지

## 5. P-GUI — 엔지니어 헤드 (P2 후 병행, P5의 전제)

**목표(SDD §13·D7):** claude-agent-sdk GUI(ClaudeApp 계열) — 하네스 project 로딩 + 승인 패널·진행 보드·게이트 결과 뷰어(표준 문서 링크)·양식형 진입점·마일스톤 알림.

**아키텍처 스케치(핵심 설계 결정 3):**
1. **GUI는 뷰 + 인간 채널만.** 상태 truth는 여전히 `events.jsonl` — GUI는 manifest·GateResult 파일의 read-model(폴링/워처). 쓰기 경로는 기존 둘뿐: 에이전트 세션(SDK 경유, 훅 게이팅 하) / 인간 승인.
2. **승인 패널 = 프로세스 분리 실행.** `pilot approve`를 **CLAUDECODE 마커 없는 별도 프로세스**로 실행해 2차 잠금과 정합 — "패널 approve 통과 + 세션 내 에이전트 approve 차단"의 동시 성립이 핵심 불변식(SDD §15 P-GUI 행).
3. **하네스 project 로딩 ON.** ClaudeApp은 격리 목적으로 `setting_sources=[]`(하네스 OFF)였다 — GUI 헤드는 반대로 리포의 CLAUDE.md·hooks·skills·.mcp.json을 **그대로 상속**해야 한다. hooks 발화·Skills 로딩·승인 채널 행동의 SDK/CLI 동등성 실증이 **착수 게이트**(§11.2 우회 스위트 재실행 포함 — SDD §15).

**구성요소 5종:** 승인 패널(카드 렌더+approve/reject) / 진행 보드(jobs 스캔·단계 상태) / 게이트 결과 뷰어(GateResult JSON + harness/standards 링크) / 양식형 진입점(`pilot new` 폼 — problem_class·mode·요청) / 마일스톤 알림(`milestone_requested` 감시).

**입구 조건:** P2 완료(보일 데이터 실존: manifest·GateResult·승인 카드·pptx). 기반 자산: ClaudeApp(H-Chat 게이트웨이·default-DENY·ClaudeSDKClient 유상태 세션 — [[claudeapp-hchat-gui-overview]]·[[claude-agent-sdk-integration-patterns]]).

**착수 체크(이미 SDD에 기록된 2건 + 1):** ① SDK/CLI 하네스 동등성 스모크(SDD §15) ② ClaudeApp default-DENY 정책 ↔ 리포 allowlist 정합(SDD §10.2 확인 항목) ③ GUI 인증 모델 결정(로컬 단일 사용자 vs 다중 — 승인 채널을 품으므로 선결).

**리스크:** SDK 하네스 동등성 미확인(스모크가 곧 게이트 — 불일치 시 아키텍처 재검토) / 승인 패널 마커 비상속 구현의 견고성 / Windows 배포 형태.

**상세계획 작성:** P2 완료 후, 착수 시점 사용자 결정.
