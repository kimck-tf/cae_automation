### 계약 1 — load_history 분석기 CLI + `load_analysis/v1` 스키마

```
python -m cae_tools.analysis.load_history --job-dir <dir> --input <req|rsp> [--input ...]
       [--gamma 0.5] [--rose-dirs 256] [--config <yaml>]
  → <job-dir>/results/load_analysis.json + <job-dir>/results/damage_rose.png
```

- 입력은 **job_dir 내부 필수**(`resolve()`+`is_relative_to` — gate-run #5 격리와 동일 원칙, 관례 위치 `jobs/<id>/inputs/loads/`). job_dir의 manifest에서 job_id를 읽어 기입(부재 시 exit 1).
- 산출 경로는 **증거 보호 경로**(§5 — 에이전트 Edit/Write 차단, 분석기 서브프로세스 파일 IO만). 분석기 자체는 훅 게이팅 비대상(SOLVER_RE 비매칭·§5 훅 표에 분석기 행 없음 — S0 상시 실행 가능), allowlist `Bash(python -m cae_tools.*)`에 포섭.
- `load_analysis.json` (G3-피로가 소비하는 공유 계약 — 키 상시 존재, 결손은 null):

```jsonc
{
  "schema": "load_analysis/v1",
  "job_id": "J-…",
  "inputs":   [{"path": "inputs/loads/case01.req", "sha256": "…", "format": "req|rsp",
                "n_channels": 3, "n_samples": 120001, "dt_s": 0.001, "provisional": false}],  // rsp=true(실샘플 검증 전)
  "channels": [{"id": "case01/fx", "source": "inputs/loads/case01.req", "column": 1, "unit": "N"}],
  "options":  {"corrections": ["none","goodman","pseudo-swt","walker"], "walker_gamma": 0.5,
               "k_set": [5, 3], "reference": {"correction": "walker", "k": 5}, "f_ult_policy": "p99.7"},
  "per_channel": [{"channel": "case01/fx", "n_cycles": 812,
                   "pseudo_damage": {"none": {"k5": 1.2e-3, "k3": 4.1e-2}, "goodman": {…},
                                     "pseudo-swt": {…}, "walker": {…}}}],
  "ranking":  [{"rank": 1, "channel": "case01/fx", "damage": 1.2e-3, "share": 0.61}],   // 기준 옵션으로 산출
  "rank_stability": {"across_corrections": true, "across_k": false,
                     "warnings": ["k=3에서 rank 1↔2 교차(case01/fx ↔ case01/fz)"]},
  "damage_rose": {"points": [{"point": "case01", "n_dirs": 256,
                              "dominant_direction": [0.71, 0.0, 0.70], "png": "results/damage_rose.png"}]},  // 3축 세트만, 스칼라 채널은 []
  "provenance": {"tool": "cae_tools.analysis.load_history", "generated_at": "…", "options_sha256": "…"}
}
```

- **지배 하중케이스 = 채널**(입력점×성분). `channels[].id`가 nCode 치환 맵·`result.json.results.fatigue.per_case[].case`와의 **조인 키**(문자열 동일성).
- pseudo-damage: 방향/채널 1D 신호 → rainflow(ASTM E1049) → 옵션별 equivalent range → `D = Σ nᵢ·Sᵢᵏ / C`(Basquin·Miner, C는 상대 스크리닝이므로 1.0 고정 — 절대 수명 아님을 docstring 명기). goodman은 f_ult 필요(p99.7 정책 — LoadVector 정책 계승)·압축평균 cycle은 보정 비활성(fallback none). walker γ=1 → none 동등(수학 항등 크로스체크 대상).
- Damage Rose: 3축 세트(t,Fx,Fy,Fz) 입력에 한해 Fibonacci 반구(기본 256 방향, **antipodal 항등 d≡−d**) 투영→방향별 rainflow→**`mean_correction="none"`(range-only) 고정** damage — 주손상 방향 산출. **none 고정 근거(C4):** −d 투영은 신호 부호를 뒤집어 cycle mean 부호가 반전되므로 보정(walker·pseudo-swt·goodman) equivalent range는 ±방향에서 달라진다 — 평균응력 보정과 antipodal 항등은 양립 불가하며, 방향 스크리닝은 none 고정이 정합하는 유일 조합(4-옵션 보정은 채널 랭킹에만 적용). `options.rose_correction: "none"`을 스키마에 명시. 스칼라 채널만이면 rose 생략(빈 배열 — 결손 아님).

### 계약 3 — G3-피로 (`gates/g3_physics/fatigue.py`, ADVISORY)

- 순수 함수(라우터 미결합으로 먼저 TDD — Task 7): `evaluate(result: dict, load_analysis: dict, criteria: dict, cases: list[dict]) -> {"verdict", "checks", "concerns"}`. 라우터 결합(Task 9)은 problem_class `fatigue_load_history` 분기에서 `results/ncode/result.json` + `results/load_analysis.json`을 로드해 호출.
- **4체크(§4.3 행 그대로):**
  1. `dominant-case-consistency` — load_analysis `ranking` 상위 K(기본 3) vs `fatigue.per_case` 손상 상위 K: 교집합 비율·1위 일치(조인 키 = 채널 id). 임계 미달 = concern.
  2. `life-order` — `fatigue.min_life_repeats`를 criteria 요구수명 오더 밴드와 대조(로그10 오더 차 임계).
  3. `damage-location` — 위치 문자열 존재 + 사례 카드 관심 위치 목록 대조. **결손 시 result "skip" 명시**(조용한 통과 금지).
  4. `case-range` — 피로 사례 카드(`response.metric: "fatigue_life"`) 범위 대조. **카드 부재 = `case-match: no_reference` check+concern**(P2 Task 13 표기 동형 — 콜드스타트 정직 표시).
- **verdict = ADVISORY 고정**(§4.3 원문 "자동 판정 없음" — FAIL 승격 없음, 이상은 concerns 축적 → M3 승인 카드 표기). **ERROR(fail-closed)만 차단**: result `state != "completed"`(P2 F2 가드 동형 — S3b는 exit_gates 없어 실패 배치도 S4로 전진하므로 이 가드가 유일 차단점) / `fatigue` 결손 / load_analysis 부재·`schema != load_analysis/v1`.
- 기준 이중 표현: `harness/criteria/g3_fatigue.yaml` ↔ `harness/standards/fatigue_standards.md`(source 인용 강제 — 기존 로더 재사용). Miner 판정선 D<1.0(무한수명 목표)/D<0.5(안전 핵심부품)를 문서화(_brain 보조 사실 — 판정선은 참고 서술, 자동 판정에 쓰지 않음).
