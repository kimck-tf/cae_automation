# P-GUI 구현 계획 — cae-copilot 엔지니어 헤드: PySide6 GUI + SDK/CLI 하네스 동등성 착수 게이트

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** SDD §13 P-GUI — claude-agent-sdk 기반 GUI 헤드(D7)를 cae-copilot 모노리포에 신설한다: 하네스 project 로딩(setting_sources=["project"]) 세션 호스팅 + 승인 패널·진행 보드·게이트 결과 뷰어·양식형 진입점·마일스톤 알림. **착수 게이트 = SDK/CLI 하네스 동등성 스모크**(§11.2 우회 스위트 SDK 경유 재실행, SDD §15) — 이 게이트를 통과해야 GUI 본체(Chunk 2)에 착수한다.

**Architecture:** GUI는 **read-model + 인간 채널**(로드맵 §5 설계 결정 3): 상태 truth는 `events.jsonl`이고 GUI는 manifest·GateResult·cards **파일만** 읽는다(D7 — 헤드 사적 상태 금지). 쓰기 경로는 기존 둘뿐 — ① 에이전트 세션(GUI가 호스팅하는 claude-agent-sdk 세션, 리포 hooks 게이팅 하) ② 인간 승인(`pilot approve`/`intervene`을 **에이전트 세션 마커가 없는 환경에서만 동작하는 분리 프로세스**로 실행(마커 감지 시 fail-closed 거부 — Codex #1) — pilot 2차 잠금 `AGENT_SESSION_MARKERS`(pilot/cli.py)와 정합). GUI는 세션 기동자로서 `CAE_JOB_ID`를 SDK 세션 env에 주입한다(§3.2). ClaudeApp의 격리 설정(setting_sources=[]·전면 default-DENY)은 **반전**한다 — 리포 settings.json allowlist가 **도구 허용의 SSOT**이고 GUI는 비허용 요청의 사람 확인 프롬프트만 담당한다(§10.2 착수 체크 ② 해소 — 프로파일 조건부 웹 deny만 옵션 조립이 담는다, 공통 계약 1).

**Tech Stack:** Python 3.11+, PySide6(Qt — optional-dependencies `gui` 그룹으로 코어와 격리), claude-agent-sdk(ClaudeApp 검증판 0.2.102 기준 — 착수 시 최신 핀 재확인), pytest + pytest-qt(offscreen). Windows 우선, 배포 = Python 앱 그대로(venv, exe 패키징 없음 — 사용자 확정).

```python
    stages: dict[str, StageView]          # status·self_corrections·gates[게이트][target]→(verdict, attempts, last_result 경로)
    milestones: dict[str, MilestoneView]  # status(approved|rejected|pending)·approver·reason·ts
    pending_milestone: str | None         # 최신 milestone_requested가 미해소(approval/rejection 없음)인 것
    advisory_flags: list[str]             # verdict ∈ {ADVISORY, NO_REFERENCE}인 (stage, gate, target) 좌표 — §6.1 패널 필수 표기
    cards: list[Path]                     # jobs/<id>/cards/*.md (seq 정렬)
    events_committed: int                 # §3.5 규약으로 읽은 커밋 라인 수(개행 종결분만)
    stale_manifest: bool                  # 이벤트/manifest 모순 감지(Codex #4) — 'pilot rebuild 필요' 신호
```

- 읽기 규약: manifest는 §3.2 스키마 기준 직접 `json.load`(원자적 rename 산출물이라 재읽기 안전), events는 **§3.5 "개행으로 종결된 라인만 커밋"** 준수 tail-read(마지막 미종결 라인 스킵), GateResult는 manifest의 `last_result` 경로로 로드. `pilot` 패키지 임포트는 **track_defs(단계 순서·게이트 정의) 로더만 허용** — store의 쓰기 함수는 임포트·호출 금지(불변식 4, Task 3 가드 테스트).
- `JobSnapshot`의 frozen dataclass **동등성(==)이 워처 변경 판정의 기반**이다(Task 4 — Codex #2): 필드를 추가할 때 동등성 의미(파생값 포함 여부)를 함께 검토한다.
- **이벤트/캐시 모순 신호(Codex #4):** `pending_milestone`(이벤트 기반)과 `milestones`(manifest 기반)가 모순되면(예: approval 이벤트 존재 ∧ manifest상 해당 마일스톤 pending) `stale_manifest=True` — pilot 쓰기 2단계(§3.5) 사이 중단의 흔적이다. 패널은 'pilot rebuild 필요' 배지를 띄우고, 해소 경로는 timeout 미확정 처리(공통 계약 2)와 동일하다(rebuild → 재조회).
