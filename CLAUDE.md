## 이 워크스페이스의 성격

코드베이스가 아니라 **리서치 산출물 워크스페이스**다. 사내 AI 공모전(결과 발표: 센터레벨 7월 중순 → 연구소레벨 8월 초)에 낼 **Hyper-Pilot**(CAE × LLM 자율 에이전트)의 전략 포지셔닝을 다듬는 한국어 분석 문서 모음이다.

- 빌드·린트·테스트가 없다. 자료는 두 묶음이다 — ① `docs/reference/`: 전략 원본(`.pdf`)과 그 메타 분석(`.md`), ② `공모전_AI로 일하는 방식/`: 공모전 공식 공지와 경쟁자 제안(이 워크스페이스의 입력·제약 자료). 실행 코드는 없다.
- **공모전 제약(`공지1.txt`)이 포지셔닝을 좌우한다:** 심사 4축(창의혁신성·실현/확대가능성·효과성·AI과제적합성), **사내 AI 도구만 허용**(H-Chat Pro·Claude Code·Copilot / 사외 도구 제외), 예측형 AI(ML·DL) 제외, 서비스화 배포본 제외.
- 상위 `0. Research/`가 git 루트다(GNN 등 여러 연구 주제 공유). 이 `7. CAE automation/` 자체는 아직 커밋되지 않은 작업본이다(`git status`상 `??`).
- 파일명 끝 6자리는 작성일 `YYMMDD`다. `260628` = 2026-06-28. 오늘 기준 절대일자로 환산해 쓴다.

## 핵심 도메인 개념

- **Hyper-Pilot = `hm-assistant`.** 자연어 → LangGraph ReAct → TCP+JSON-RPC → HyperMesh `hwx` Python API **실시간 원격제어** + **물리 이론값 기반 검증 가드레일**을 갖춘 자율 CAE 에이전트. 전략 원본 3종의 중심축은 "상용 전처리기를 라이브 GUI 세션으로 직접 제어 + 결과 검증"이라는 희소 영역(소켓/RPC ~5%)을 선점했다는 주장이다.
- **자료의 핵심 대립축 = RPC vs MCP.** 원본 3종은 RPC 직접제어를 독보적 해자로 본다. `Critical_Review`는 이를 **과장으로 판정**한다: ① MCP도 내부적으로 JSON-RPC 2.0이라 "RPC의 반대편"이 아니라 표준 도구계층이다, ② Abaqus-Control-MCP 등 이미 라이브 GUI 세션을 제어하는 MCP 사례가 있어 "유일성" 주장이 약하다, ③ 결론은 **"내부는 전용 RPC 세션엔진, 외부는 MCP 어댑터"의 하이브리드**이며 진짜 해자는 RPC가 아니라 *상용 전처리 워크플로를 끝까지 신뢰성 있게 완결하는 도메인 엔진 품질 + 이론값 검증*이다. (원본 3종도 말미 로드맵에서 MCP 어댑터 추가를 권고 → 두 문서의 결론은 수렴한다.)
- **"이론값 기반 가드레일"은 가장 강한 주장이자 가장 약한 증거다.** 선형 빔·판·브라켓처럼 이론값 비교가 쉬운 문제와, 비선형·접촉·충돌·열-구조 연성처럼 이론값이 모호한 문제는 검증 방식이 달라야 한다. "만능 검증기"로 보이게 하지 말고 문제유형별 레이어로 한정해 말한다. 정량 지표(성공률·개입횟수·오결과 차단율 등)가 아직 비어 있다는 게 공통 약점.

## 작업 규약

- **정직성·포지셔닝 규칙을 지킨다.** "세계 최초", "범용 보고서 자동화", "MCP가 차별점", "RPC가 유일한 해자", "이론값 가드레일은 만능" 같은 광범위 주장은 선례·반증과 충돌하므로 쓰지 않는다. 새 주장에는 충돌 선례와 방어선을 함께 적는다.
- **원본(PDF)과 분석(MD)의 층위를 섞지 않는다.** `Summary`는 원본을 충실히 요약하는 문서, `Critical_Review`는 외부 근거로 반박하는 문서다. 원본 주장을 분석 문서의 확정 사실처럼 인용하지 말고, 어느 층위의 진술인지 밝힌다.
- **외부 인용은 1차 출처에서 확인한 것만 쓴다.** `Critical_Review`의 URL 각주(MCP 문서·arXiv·GitHub)가 그 기준이다. 새 좌표·URL을 추가할 때 같은 기준을 지킨다(추측 인용 금지).
- **경쟁자를 의식해 차별점을 좁힌다.** `예시케이스1·2`도 자연어 지시 → 메시·해석·보고 전 과정 자동화 + 경험 자산화를 내세운다(예시1은 구현 검증까지 주장). 따라서 "자동화 자체"는 차별점이 아니다. Hyper-Pilot 고유 방어선(상용 전처리 워크플로 완결성 + 이론값 검증 가드레일)으로 한정해 말한다. **사내 도구 한정 규정과 LangGraph(사외 프레임워크) 스택의 정합성**은 열린 확인 대상이다.
