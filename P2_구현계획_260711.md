## 공통 계약 (본 계획의 SSOT 발췌)

**러너 CLI(§7.2 — track1.yaml `solver_run` 패턴과 일치 필수):**

```
python -m cae_tools.runners.<solver> submit  --deck <path> --job-dir <dir>   → job_ticket.json (즉시 반환)
python -m cae_tools.runners.<solver> status  --job-dir <dir>                 → {"state": "running|completed|failed"}
python -m cae_tools.runners.<solver> collect --job-dir <dir>                 → result.json + extracted/*
```

- 산출 위치: `<job-dir>/results/solver/`(job_ticket.json·result.json·extracted/) — **에이전트 쓰기 차단 경로(증거, §5)**. 러너는 서브프로세스 파일 IO라 무영향(§10.2). 디렉터리명은 **리터럴 `solver/`**(트랙① = 1 job 1 솔버 — 다중 러너 디렉터리는 P4 nCode에서 재검토).
- `job_ticket.json`: `{"job_id", "solver", "deck", "pid", "submitted_at"}` — collect가 `job_id`를 manifest와 대조(오귀속 방지 §3.2).
- **detach**: Windows `creationflags=DETACHED_PROCESS|CREATE_NEW_PROCESS_GROUP`, POSIX `start_new_session=True`.
- **솔버 커맨드 주입**: 실행 커맨드는 환경변수 `CAE_SOLVER_CMD_<SOLVER>`로 재정의 가능(기본 = 실 솔버 커맨드) — 테스트는 mock 솔버 스크립트를 주입한다. 주입은 "무엇을 실행하나"만 바꾸고 계약(티켓·상태·수집)은 동일 코드가 수행한다.
- `result.json` 공통 스키마(§7.2 원문 그대로): `solver/state/exit_code/wall_time_s/error{family,code,signature,log_excerpt}/results{max_stress,max_displacement,reactions{sum,applied},extracted_files}` — **`reactions`는 G3 평형 잔차의 필수 입력**.

**G3 판정 모드(§4.3):** closed_form=이론값 자동 PASS/FAIL / linear_static_complex=평형잔차·단위·오더·패턴+실적범위 **ADVISORY** / fatigue(P4)·nonlinear=ADVISORY(참고 신호만). ADVISORY는 비차단 — M3 승인이 최종.
